作战地图 - 项目技术文档
版本: 2.0
最后更新: 2026-05-15
用途: 考研复习（或其他个人成长）可视化管理系统，以文明游戏风格的地图占领与战斗机制，追踪学习进度与领土扩张。

1. 项目概述
“作战地图”是一个单文件 HTML 应用，无需任何服务器或构建工具，直接在浏览器中运行。它将学习过程模拟为战略游戏：

左侧军种列表代表不同学习科目（可自定义名称、颜色、备注）。

中央六边形网格地图可导入底图，每一格可被军种“占领”或经历“战斗”。

右侧提供军队详情、地块详情编辑面板。

支持管理员模式（配置军种、编辑地块属性）与用户模式（记录学习时长、占领地块、进行战斗）。

数据通过 localStorage 与 IndexedDB 持久化，并支持导出/导入完整备份（包含底图）。

2. 整体架构
2.1 布局结构（CSS Grid）
页面使用 grid-template-areas 将可视区域划分为：

区域	用途
top-bar	顶部状态栏：倒计时、总学习时长、专注度滑块、精力/士气占位、退出按钮
left-panel	左侧军种列表（可滚动，管理员可添加新军种）
map-area	中央地图画布（Canvas）
right-panel	右侧详情面板（军队详情 + 地块详情）
bottom-bar	底部工具栏（视图重置、地图编辑按钮、坐标显示、缩放按钮）
四角 (top-left, top-right, bottom-left, bottom-right)	放置模式相关按钮（管理员模式切换、导入底图、导出备份等）
2.2 技术栈
HTML5 Canvas：绘制六边形地图、底图、交互反馈。

CSS Grid + 自定义属性：复古文明风格UI。

Vanilla JavaScript：所有逻辑，无框架依赖。

localStorage：存储军队列表、六边形属性、日志、焦点值。

IndexedDB（未在最终版本中完全使用，但架构预留）：存储底图图片。

FileReader API：导入/导出备份 JSON 文件。

3. 核心功能模块
3.1 地图与六边形系统
坐标系统
使用 Axial Coordinate（轴向坐标）(q, r) 表示每个六边形。

axialToPixel(q, r) 将轴向坐标转换为画布像素坐标。

地图尺寸定义为 A0_WIDTH = 3370, A0_HEIGHT = 2384（A0 分辨率）。

六边形半径 HEX_RADIUS = 60，水平间距 SQRT3 * R，垂直间距 1.5 * R。

六边形生成 (generateHexForBounds)
传入一个世界边界 {minX, minY, maxX, maxY}，计算覆盖该矩形区域的所有轴向坐标范围。

遍历 q, r，跳过已存在的 hexData 条目，创建新的六边形数据对象。

每个六边形对象包含：

javascript
{
  q, r,             // 坐标
  armyId: -1,       // 占领的军种索引（-1 表示未被占领）
  customName: '',   // 自定义名称
  adminColor: '',   // 管理员赋予的属性颜色
  battleStartTime: null, // 战斗开始时间戳
  battleArmyId: -1       // 发动战斗的军种索引
}
多区域（A0）拼接
regions 数组存储每个 A0 区域的位置 {x, y, width, height} 及其底图 Image 对象。

初始只有一个区域 {x:0, y:0}。

管理员通过“地图编辑”模式，点击区域边缘的加号按钮，可向上下左右添加新的 A0 区域。

添加新区域后，调用 generateHexForBounds(getWorldBounds()) 扩展六边形网格。

3.2 军队（军种）管理
armies 数组存储所有军种，每个对象包含：

javascript
{
  name: '',          // 军队名称
  note: '',          // 备注（如“英语阅读”）
  color: '',         // 颜色（十六进制）
  totalMinutes: 0,   // 累计学习分钟数
  todayMinutes: 0    // 今日学习分钟数
}
左侧面板渲染军种卡片，点击选中。

管理员模式下可编辑名称、备注、颜色；底部出现“+ 新建军种”按钮，可动态增加军种。

用户模式下选中军种后，可记录学习时长，并用于占领/战斗。

3.3 用户/管理员双模式
切换按钮位于右下角，点击后 adminMode 翻转。

用户模式：

左侧选择军种，右侧显示“今日学习”输入框和“计入”按钮。

选中军种后点击空地 → 直接占领（无属性颜色时）；若该地块已被管理员设置了属性颜色，则进入“战斗”状态。

未选中军种时，右侧显示“全军概况”。

左侧不显示新建军种按钮。

左下角按钮组为“导入备份”和“导出备份”。

管理员模式：

左侧可编辑军种名称、备注、颜色，并新建军种。

右侧军队详情中，学习时长输入框被禁用；可修改军队颜色。

点击地块不会改变占领状态，但可编辑地块的“属性颜色”和自定义名称。

左下角按钮组变为“导入底图”和“清除底图”。

地图上方显示红色“管理员模式”提示。

底部出现“地图编辑”按钮，开启后可拼接 A0 区域。

3.4 地块占领与战斗
占领逻辑
用户模式 + 已选军种 + 点击无属性颜色的空地 → armyId = 选中军种索引，customName = 军种名称。

若地块已有属性颜色（adminColor 非空）且未被占领 → 进入战斗：设置 battleStartTime 和 battleArmyId，取消军种选中。

战斗机制
战斗中的地块在地图上呈现呼吸动画：属性颜色与军种颜色交替闪烁，边框呼吸高亮（军种颜色）。

用户点击战斗中的地块，右侧出现“胜利”/“失败”按钮。

胜利：armyId 设为战斗军种，清除战斗状态，记录日志。

失败：保持属性颜色，清除战斗状态，记录日志。

自动超时：若 3 天内未手动处理，自动判定失败，清除战斗状态。

取消占领
用户模式未选中军种时，点击已占领地块，右侧出现“取消占领”按钮。

执行后 armyId = -1，地块恢复为未占领状态，若有管理员属性颜色则显示该颜色，否则变回空地。

3.5 学习时长记录
用户模式 + 已选军种，输入小时数，点击“计入”按钮。

累加 totalMinutes 和 todayMinutes，并写入 studyLogs 日志。

顶部“总时长”实时更新。

3.6 数据持久化与备份
自动保存
每次修改军队、六边形属性、日志后，调用 saveAllData() 将全部数据序列化存储到 localStorage：

examHexData：六边形属性映射 { key: {q, r, armyId, customName, adminColor, battleStartTime, battleArmyId} }

examArmies：军队数组

examFocus：专注度数值

examRegions：区域坐标数组

actionLogs：操作日志

studyLogs：学习日志

导出备份
点击“导出备份”按钮，收集所有数据（包括当前显示的底图转为 Base64），生成一个 JSON 文件下载。

导出格式详见第 5 节。

导入备份
选择之前导出的 JSON 文件，解析后覆盖当前所有数据（军队、六边形、日志、底图等），并重绘界面。

3.7 交互与视图控制
平移：鼠标左键拖拽画布。

缩放：鼠标滚轮或底部 +/- 按钮，显示百分比。

倾斜视角：鼠标右键上下拖拽改变俯仰角（Y 轴压缩）。

重置视图：回到地图中心，俯仰角归零，缩放自适应。

提示框：鼠标悬停在六边形上时，显示地块信息（名称、所属军队、坐标），离开画布或进入 UI 区域时自动隐藏。

5. 导出数据格式（JSON）
导出的 .json 文件结构如下：

json
{
  "version": 2,
  "time": "2026-05-15T12:00:00.000Z",
  "armies": [
    {
      "name": "政治",
      "note": "政治理论",
      "color": "#e74c3c",
      "totalMinutes": 120,
      "todayMinutes": 30
    }
  ],
  "focus": 75,
  "regions": [
    { "x": 0, "y": 0 }
  ],
  "hexProps": {
    "q,r": {
      "q": 0,
      "r": 0,
      "armyId": 2,
      "customName": "极限题集",
      "adminColor": "#ffaaaa",
      "battleStartTime": null,
      "battleArmyId": -1
    }
  },
  "bgImages": {
    "0": "data:image/jpeg;base64,..."
  },
  "actionLogs": [
    {
      "time": "2026-05-15T10:30:00.000Z",
      "type": "占领",
      "army": "数学",
      "coords": "(5, -3)",
      "result": "成功"
    }
  ],
  "studyLogs": [
    {
      "time": "2026-05-15T11:00:00.000Z",
      "army": "英语",
      "hours": 1.5
    }
  ]
}
字段说明：

armies：军种数组，包含当前名称、颜色、累计及今日学习分钟数。

hexProps：以 "q,r" 为键的六边形数据，注意 battleStartTime 为时间戳或 null。

bgImages：键为 regions 的索引（字符串），值为对应 A0 区域的底图 Base64（JPEG）。

actionLogs 与 studyLogs：操作日志与学习明细，均为数组。

导入时，脚本会解析该文件，并覆盖 armies、hexData、regions、focus、日志等，同时根据 bgImages 恢复底图。

6. 注意事项与后续扩展
兼容性：依赖现代浏览器的 Canvas、localStorage、FileReader、IndexedDB（可选），建议使用 Chrome/Edge 最新版。

性能：六边形数量随 A0 区域增加而增长，目前一个区域约几百个六边形，性能良好。若区域过多，可考虑视口裁剪优化（已实现视口内渲染）。

精力/士气值：当前为占位符，计划未来关联学习时长或战斗结果。

多设备同步：目前仅支持单设备本地存储，导出/导入可实现手动迁移。

数据安全：清除浏览器数据会丢失所有进度，请定期导出备份。
