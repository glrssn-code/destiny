# Destiny - 作战地图

## 项目概述
- **名称**: 作战地图（Battle Map）
- **用途**: 考研复习可视化管理系统，以文明游戏风格的地图占领与战斗机制追踪学习进度
- **类型**: 单文件 HTML 应用，无需服务器，直接在浏览器运行
- **仓库**: https://github.com/glrssn-code/destiny.git

## 技术栈
- HTML5 Canvas（绘制六边形地图）
- CSS Grid（复古文明风格 UI）
- Vanilla JavaScript（无框架依赖）
- localStorage（数据持久化）
- FileReader API（备份导入/导出）

## 核心常量
- 地图尺寸: A0_WIDTH=3370, A0_HEIGHT=2384
- 六边形半径: HEX_RADIUS=60
- 目标日期: 2026-12-26（倒计时）
- 默认军种: 政治、英语、数学、专业课、健身、阅读

## 核心功能
1. **军种管理** — 左侧面板，管理学习科目（管理员可添加/编辑）
2. **六边形地图** — 中央画布，支持底图导入、多区域拼接
3. **双模式**: 用户模式（记录学习/占领/战斗）、管理员模式（配置管理）
4. **战斗机制** — 占领属性颜色地块触发战斗，3天超时自动失败
5. **数据备份** — JSON 格式导出/导入（含底图 Base64）

## 关键文件行数参考
- 1-200: HTML/CSS 布局与样式
- 200-400: 布局结构与组件
- 400-550: JavaScript 初始化与常量
- 550-730: 坐标转换、六边形生成、区域管理
- 730-870: 数据持久化（日志、localStorage）
- 870-960: 军队详情面板
- 960-1150: 动画渲染（呼吸效果）
- 1150-1350: 鼠标交互（拖拽、缩放、倾斜）
- 1350-1600: 备份导入/导出、视图控制

## 注意事项
- 依赖现代浏览器（Chrome/Edge）
- 定期导出备份防止数据丢失
- 目前仅支持单设备本地存储
